## 1. Visual Identity Statement（视觉身份宣言）

### 🎯 视觉核心句

> **"画面要让一个真实的中国白领，瞄第一眼就觉得'这就是我办公室'——然后第二眼，才发现哪里不对劲。"**

第一眼是**认同**，第二眼是**刺破**。认同感是诱饵 —— 它把玩家拉进来，才能在第二眼用视觉幽默/压迫/荒诞刺到玩家。这句话是所有视觉决策的第一道门槛：任何画面如果第一眼不像真实办公室、或第二眼没有刺点，就是未达标。

### 🧱 原则 1：格子压人

**定义**：工位网格与物件堆叠共同制造"空间被榨干"的视觉压迫感，让玩家在看到画面构图时就感受到"余量不足"。

**设计检验**：当某个场景"感觉太宽敞"时，这个原则说：加一排工位、加杂物、压低走道宽度，直到"呼吸感"消失。

**服务的 Pillar**：#1 平庸是一种艺术 —— 密集的格子直观传达"每个人都被钉在自己的方格里，无处突破也无处逃跑"。

**具体示例**：玩家工位 sprite 的"空地面积"不超过整块格子的 15%，桌面永远有水杯、外卖袋、或绿萝遮角，不留视觉留白。

### 🔦 原则 2：时钟光语

**定义**：光源种类和色温映射当前时段与玩家精神状态 —— 日光灯惨白（常规工时）、电脑屏蓝光叠加（加班）、窗外日落橙（今天撑过去了），每种光态都是一帧叙事。

**设计检验**：当一个新场景"看不出是几点"时，这个原则说：指定一个主光源并写进 scene 备注，光态不明确则场景情绪缺失。

**服务的 Pillar**：#2 叙事即机制 —— 玩家不需要看 HUD 上的时间数字，光线本身就是剧情进度条。

**具体示例**：同一张工位 palette，白天版主色为 `#D4D0C8`（日光白），加班版叠加 `#1A2A4A` 蓝偏；美术需输出两套光态 sprite variant，命名后缀 `_day` / `_overtime`。

### 😵 原则 3：喜丧美学

**定义**：高压事件（被骂、KPI 暴涨、猝死预警）用夸张色块和漫画式符号呈现，而非写实恐惧 —— 把痛苦包装成段子的视觉形式，笑容和绝望共用同一张脸。

**设计检验**：当一个负面事件的视觉反馈"让玩家感到焦虑而非失笑"时，这个原则说：色调偏冷/暗过头，改用高饱和撞色 + 漫画字效（"!!!"符号），保持荒诞感。

**服务的 Pillar**：#4 苦中作乐 —— 黑色幽默不能只靠台词，视觉本身必须会"笑着刀人"。

**具体示例**："被 PUA 对话"触发时，Boss 的头顶出现一个金色光圈 sprite（圣人光环），文字框边框变成励志鸡汤配色（暖黄 `#F5C518`），但内容是压迫性话术 —— 视觉与内容的错位制造黑色幽默。

### 📜 视觉身份宣言段落

> 《活过第 X 集》的视觉语言从一个承诺出发：**第一眼认出，第二眼刺破**。画面以 SFC 级像素精度还原中国开放式办公室的物质真实 —— 密集的工位格子、惨白的日光灯、堆满桌面的外卖袋与绿萝 —— 让白领玩家在视觉上"到家"。但这份认同感是诱饵：光线会像剧情进度条一样推进，加班的蓝光压着你；每一次负面事件都用喜丧美学包裹，把结构性痛苦画成段子，笑着递到玩家手里。这款游戏的像素不追求怀旧，追求的是"这个地方我认识，但我在这里活不长"的那种冷静清醒。

## 2. Mood & Atmosphere（情绪与氛围）

按游戏状态定义情绪目标 —— 具体到灯光师能直接据此工作的精度。**MVP 关键状态 6 个**全量定义，non-MVP 状态一行备注。

### 2.1 主菜单 / 标题画面

| 维度 | 细节 |
|---|---|
| **情绪** | 像周一早上刷打卡记录，已经迟到但还在往里走 |
| **光线** | 清晨 6:58，日光灯冷白（5800K），对比度低 —— 一切还没开始 |
| **描述词** | 惺忪、将就、例行 |
| **能量** | stagnant |
| **必备视觉** | 未亮满的日光灯 pixel 闪烁动画；工位排列从黑暗渐显 |
| **错位设计** | logo 字体用公文宋，但"第 X 集"的 X 是手写涂改液覆盖 —— 官方感里藏着劫数 |

### 2.2 早晨事件预告

| 维度 | 细节 |
|---|---|
| **情绪** | 像看到今日排班表，心里已经开始盘算怎么苟过去 |
| **光线** | 日光灯全亮，白冷均匀，无阴影 —— 信息裸露 |
| **描述词** | 板正、预谋、沉默 |
| **能量** | measured |
| **必备视觉** | 仿报纸版式卡片；预告图用单色网点印刷质感 |
| **错位设计** | 卡片设计像公司内刊，但标题写的是"今日生存概率：64%" |

### 2.3 白天行动阶段

| 维度 | 细节 |
|---|---|
| **情绪** | 像在会议室喝第三杯咖啡还是清醒不了 |
| **光线** | 日间日光灯 + 屏幕蓝光叠加，色温混乱（4200K 暖冷打架） |
| **描述词** | 拥挤、将信将疑、钝感 |
| **能量** | measured → stagnant（随 AP 消耗推移） |
| **必备视觉** | 行动卡堆叠在屏幕蓝光溢出的工位上；NPC 表情仅 3 帧循环 |
| **错位设计** | 行动卡图标精致如 RPG 技能，但名字是"发一封没人回的邮件" |

### 2.4 下班抉择节点 ⭐

*游戏的心理签名时刻。所有视觉语言在这里达到最大张力。*

| 维度 | 细节 |
|---|---|
| **情绪** | 像电梯门已经开了，但你的腿没动 —— 不是不能走，是不确定走不走得了 |
| **光线** | 日落橙斜射入窗（3000K），长影打在工位上；屏幕蓝光与暖光对峙 |
| **描述词** | 拉锯、窒息、将燃未燃 |
| **能量** | frenetic（内心）/ stagnant（画面静止）—— 两层同存 |
| **必备视觉** | 时钟指针在 18:00 上颤动（pixel 抖动动画）；三张选项卡用阴影厚度区分心理压力 |
| **错位设计** | **加班选项卡边框发光，像商场促销标签**——"划算"的错觉通过 UI 语言植入 |

### 2.5 今日 / 周五总结

| 维度 | 细节 |
|---|---|
| **情绪** | 像地铁上刷今天的行程记录，花了多少不重要，重要的是确实活过去了 |
| **光线** | 夜间地铁车厢灯（暖黄 3500K），窗外黑 —— 内外隔绝 |
| **描述词** | 倦意、间歇性满足、收尾 |
| **能量** | contemplative |
| **必备视觉** | 数值用收据热敏打印字体滚动显示；"本周有意思的事"用便利贴手写体 |
| **错位设计** | **收据最底部印"感谢您的付出，请问下次还来吗？"**——问候语格式，内容是威胁 |

### 2.6 月末 KPI 考核 → GAME OVER

| 维度 | 细节 |
|---|---|
| **情绪** | 像连续剧最后一集片尾字幕跑完，你还坐在沙发上没动 —— 知道结束了，但没准备好 |
| **光线** | 全屏骤降至深蓝灰（2200K），唯一光源是考核结果框的红光溢出 |
| **描述词** | 仪式感、讽刺、尘埃落定 |
| **能量** | contemplative（前段）→ frenetic（结果揭晓瞬间） |
| **必备视觉** | KPI 表格逐行"打印"出来；GAME OVER 字样用公文落章 stamp 样式覆盖 |
| **错位设计** | **画面底部出现"第 X 集 完"字幕 + 片尾曲进度条**——死亡被包装成播出事故 |

### Non-MVP 状态（一行备注，Phase 后期再展开）

- **周末场景**：用暖黄低饱和度区隔工作日，但总有一条消息提示不期而至
- **周年庆事件**：全屏拉花彩带特效，KPI 乘以 1.5 —— 节日即陷阱
- **事件 CG（关键剧情插画）**：跳出 pixel 格，用仿四格漫画黑白线稿强调叙事断点

## 3. Shape Language（形状语言）

SFC 级像素可承载比 NES 更细腻的剪影（子像素暗示、精细明暗），形状语言要匹配这个精度。所有形状决策必须回溯到 Section 1 原则或 Pillar。

### 3.1 角色剪影哲学

- **辨识度第一原则**：所有 NPC 剪影必须在 32×64 px 缩略图下、**不依赖颜色**即可区分。辨识依赖**重心位置**与**轮廓极值点**（最高/最宽处的特征形状）。
- **夸张系数 1.5×**：姿态比现实略超，但不进入 chibi 比例；头身比约 1:4.5。
- **NPC 原型剪影特征**：

| 原型 | 剪影关键特征 |
|---|---|
| **卷王** | 脊背 100% 垂直，头顶 2 px 竖立发丛，颈部前伸直角 —— 剪影顶端"¬"钩形 |
| **摆烂族** | 重心下坠，肩线单侧低 4 px，躯干向椅背倾 15° —— 剪影底部 > 顶部 |
| **老油条** | 左臂外展持茶杯（3 px 圆形突出），站姿微弓背 —— 剪影中段侧向突起 |
| **谄媚族** | 头前倾，双手腹前交叠 —— 剪影"蛋形"无锐角（人畜无害包装 / 呼应 Pillar 4）|
| **新人** | 直挺但拘谨，双手紧贴躯干 —— 剪影窄长、无突出物 |

### 3.2 环境几何

- **主导形式：正交硬线格** —— 工位区由 16×16 px 模块组成严格网格；水平/垂直绝对正交，无有机曲线。**直接实现 Section 1 原则 #1"格子压人"—— 环境几何即权力结构**
- **空间权力语言**：格子密度与权力成反比 ——
  - 工位区：8 px 间距（密集）
  - 走道：16 px（中等）
  - 老板办公室：无格线，大面积实心矩形（越靠近权力中心，几何越稀疏）
- **茶水间**：唯一 4 px 圆角区域 —— 全图最"柔软"的空间，非正式社交场是逃逸口
- **异常几何信号**：危机事件触发时，格线局部倾斜 2-3° —— "电梯门快关上"的微妙错位感，不破坏整体正交
- **具体像素决策**：工位隔板顶端常态 1 px 直线；"即将被炒"状态触发时，该线段出现 1 px 断口 —— 细微但可读

### 3.3 UI 形状语法

- **UI 母题：公司内刊 + 打卡机界面** —— 所有面板使用 **0 圆角直角矩形**，边框线 1 px，模拟复印机打出的表格质感
- **对话框**：无气泡，改"报告格式框" —— 顶部 8 px 灰色标题栏（公文抬头风）；与环境几何同族，不跳出世界观
- **数值框（精力 / KPI / 关系）**：横向进度**表格**，格子逐格填色 —— 像 Excel 单元格而非传统血条
- **按钮**：直角；hover 时边框变双线（1 px 内 + 1 px 外），模拟盖章效果；禁用渐变与圆角
- **UI 几何 = 环境几何同构**（都是正交直角）—— 强化"你就活在这张表格里"的压迫感

### 3.4 Hero Shapes vs Supporting Shapes

- **Hero 法则**：关键交互对象（可点击 NPC、重要道具、事件触发点）使用**孤立轮廓** —— 周围留 2 px 空气感间距，与环境格线不接触；视觉"浮起"
- **Supporting 法则**：背景工位/地板格/墙饰**贴合网格、无内部细节**，主动退后
- **可交互 NPC 脚下 2 px 实线底影**（非渐变阴影）—— 这是全画面**唯一水平实线不属于格线系统**的元素，视觉上成为"请注意我"的信号
- **危险时刻的形状破格**：高压事件（Boss 呼叫、绩效预警）时，事件源 NPC 周围产生 3 px 不规则抖动轮廓 —— 几何破格即叙事语言（呼应 Pillar 3 死亡是注定的 —— 破格即预兆）
- **黑色幽默形状包装**：严肃正经的公文框几何 + 荒诞内容文字（如绩效评分表里写"存活率：72%"）—— 形式与内容的落差是喜丧美学的形状级表达

## 4. Color System（色彩系统）

SFC 技术约束：每 sprite 最多 16 色调色板（含透明），全屏同时 256 色（多调色板），支持半透明叠加。

### 4.1 主调色板（World Palette）

| 色名 | Hex | 本作专属语义 | 使用场景 |
|------|-----|------------|---------|
| 打工人黄 | `#C8A85A` | 荧光灯把皮肤打成这个色 —— 不是阳光，是"被照"；活着但不发光 | 皮肤高光、工位桌面、考勤打卡机 |
| 格子间灰蓝 | `#5A7080` | 隔断板的颜色；边界即压迫，见色即想低头 | 隔断墙、走道地板、UI 背景底色 |
| 档案室棕 | `#7A5838` | 堆了三年没人翻的文件夹；沉默的证人 | 桌面旧文件、老板桌、抽屉 |
| 白炽灯白 | `#E8E0CC` | 不是干净，是漂白；加班到天亮的颜色 | 天花板、办公纸张、对话气泡底 |
| 屏幕蓝光 | `#2C4A6E` | 唯一来自"外面"的光，却是电子牢笼；越加班越深 | 加班时全屏叠层、电脑屏幕、overtime 光态变体 |
| 地铁座椅橙 | `#B05A28` | 下班后唯一属于自己的 5 分钟 | 地铁场景、今日总结背景、片尾字幕条 |
| 咖啡渍棕黑 | `#2A1F14` | 杯底剩下的；熬过去了但留下印子 | 阴影底色、角色轮廓线、GAME OVER 底层 |

> **禁用纯黑 `#000000`** —— SFC 用 `#2A1F14` 作最深色，避免美工刀感。

### 4.2 语义色词典（Semantic Color Vocabulary）

| 色类 | 本作专属意义 | 反例（何时不用它表达该意义） |
|------|------------|--------------------------|
| **红** | KPI 超标警告 + 领导印章红 + 喜庆俗艳（**同一个红，语境决定悲喜**） | 不用红表示"受伤/血量下降" —— 本作无 HP 血条 |
| **黄** | 疲倦的存在感；不是警告，是"我还活着"的证明 | 不用黄表示"胜利/成就" —— 那个位置属于老板金 |
| **蓝** | 屏幕囚禁色；光源但是陷阱，越亮越深陷 | 不用蓝表示"平静/可信赖" —— 本作蓝是异化，不是安全 |
| **绿** | 绩效表格里的"达标绿"；被允许存在的颜色，**但绝不意味着好** | 不用绿表示"自然/治愈" —— 本作无自然场景 |
| **紫** | 缺席的颜色；老板位椅背偶尔一抹，身份压迫不需要频繁出现 | 不用紫表示"神秘/魔法" |
| **白** | 被漂白的，不是纯净；荧光灯下一切都失真 | 不用白表示"胜利/结局良好" |
| **黑** | 咖啡渍的底；熬过了但不会消失的痕迹 | 不用纯黑 `#000000` |

### 4.3 区域色温规则（Area Color Temperature）

| 场景 | 基础色温 | 主色倾向 | _day / _overtime 变体规则 |
|------|---------|---------|--------------------------|
| 工位区 | 4200K | 打工人黄 + 格子间灰蓝 | `_day`：黄主导；`_overtime`：屏幕蓝光叠 30% 降饱和 |
| 走道 | 4800K | 白炽灯白 + 档案室棕阴影 | `_overtime`：阴影拉长，棕加深至 `#5A3A20` |
| 茶水间 | 3200K | 地铁座椅橙暖调 | **无 `_overtime` 变体** —— 茶水间是逃逸锚点，色温固定 |
| 老板办公室 | 3000K | 档案室棕 + 紫边光 | `_day`/`_overtime` 均维持 —— 老板时间不分昼夜 |
| 会议室 | 5500K | 白炽灯白主导，蓝光投影 | `_overtime`：蓝光比例升至 60%，角色脸部失真 |

### 4.4 UI 调色板 vs 世界调色板

**判断：UI 使用独立子调色板，但色值必须从世界调色板派生。**

理由：SFC 16 色/sprite 约束下，世界 sprite 与 UI sprite 共用调色板会导致色槽耗尽；独立子板保留扩展空间，同时维持视觉一致性。

| UI 元素 | Hex | 派生逻辑 |
|---------|-----|---------|
| 主按钮底色 | `#5A7080` | 格子间灰蓝 —— UI 是格子间延伸 |
| 按钮 hover | `#7A96A8` | 灰蓝提亮 20%，仍在同色系 |
| 框线 / 边框 | `#2A1F14` | 咖啡渍棕黑 —— 一切边界都是痕迹 |
| 正文文字 | `#E8E0CC` | 白炽灯白 —— 可读但失真 |
| 高亮 / 焦点 | `#C8963C` | 老板金 —— 焦点即权力 |
| **UI 警告黄** | `#F5C400` | 色盲安全高对比黄 —— UI 系统层警告专用 |
| 世界警告红 | `#C83428` | 仅在**世界道具**（KPI 印章、考核表、庆典俗艳）使用；**UI 系统层不再用此色作警告** |
| 系统提示背景 | `#1A2A38` | 屏幕蓝光加深 —— 系统 = 囚禁 |

**UI 叠加方式：灰度叠加世界底层（multiply blend）** —— UI 不应"飘"出世界，它是世界的一部分。

### 4.5 色盲与可达性安全

**优先关注**：红绿色盲（Deuteranopia / Protanopia，中国男性约 5%）+ 全色盲辅助（像素艺术高对比）

| 失效色对 | 色盲类型 | 失效后果 | Backup 方案 |
|---------|---------|---------|------------|
| KPI 警告红 `#C83428` vs 达标绿 `#4A7840` | 红绿色盲 | 两色等亮度灰值，无法区分绩效状态 | **形状补偿**：红=下箭头，绿=上箭头；**纹理补偿**：红格子1px 斜线 / 绿实心 |
| 老板金 `#C8963C` vs 打工人黄 `#C8A85A` | 红绿色盲轻度 | 饱和度接近，权力感消失 | **明度拉开**：老板金提亮 `#E0B050`，打工人黄压暗 `#A88840`；加边光描边 |

**案例 1 —— 绩效单 KPI 格**：红格内绘制 1px 斜线纹理（hatching），绿格保持纯色填充。
**案例 2 —— 领导印章**：2px 双线描边（非纯色）；印章内加汉字"批"字图案作图标层。

### 4.6 点缀色（Accent Palette）

| 色名 | Hex | 语义 | 使用预算 | 使用规则 |
|------|-----|------|---------|---------|
| 老板金 | `#E0B050` | 权力的颜色；稀有才有压迫感 | ≤ 3% 全画面像素 | 仅用于老板相关 UI/物件；禁止用于普通交互元素 |
| 周年庆俗艳粉 | `#E8609A` | 公司庆典的廉价喜气；喜丧美学爆点 | ≤ 5%，且**必须搭配黑框线压住** | 仅出现于"庆典演出"场景；日常画面禁用 |
| PUA 鸡汤暖金圈 | `#C8A028` | 激励标语的颜色；越金越空洞 | ≤ 2%，仅图标级别 | 出现即**必配讽刺文本**；禁止单独作正面强调 |
| 片尾曲红 | `#E03020` | 结局已定；不是血，是落幕字幕 | ≤ 8%，仅 GAME OVER / 结局屏 | 全游戏流程中最后出现；**出现即不可撤销** |

## 5. Character Design Direction（角色设计方向）

### 5.1 玩家角色（老油条）视觉原型

**可见时机**：主视角 2.5D 侧视角工位全景（详见 5.4）。**玩家角色全程可见于左下工位格**，但镜头中心永远在目标 NPC/事件上 —— 玩家处于视野边缘，"存在感刚够，不抢戏"。互动时切入**半身特写**（对话框上方）。

**投射哲学**：玩家是"你的替身"，设计刻意留白 —— 无发色设定（默认黑；灰 / 棕可选）、无明确性别特征、西装标准款。**个性来自状态叠加，不是外貌**：
- 加班叠 3 层 → 领带松动 + 1px 眼袋
- 摸鱼超阈值 → 手机在桌面下的阴影

**标志性剪影元素**（承接 Section 3）：
- **茶杯永挂左手**，3px 凸起是唯一轮廓突出点
- 背微驼（非过分），视觉重心压低，"已经在这待了很久"
- 头发无高光 ——"油"是姿态的油，不是头发油

**"老"的视觉语言**：肘部磨损纹路（2px 深色描边）、鞋底阴影比新人厚（+1px）、走路步幅小、无跳跃感
**"油"的视觉语言**：表情默认 0.5 眯眼（不关心但在看）、碰危险第一反应是缩肩而非挺胸

### 5.2 NPC 原型辨识规则（MVP 9 原型）

每个 NPC 都要有"喜丧触发"—— 这是视觉载荷的黑色幽默。

| 原型 | 剪影关键特征 | 默认 + 变体 | 个人化细节 | **喜丧触发** |
|------|------------|------------|-----------|-------------|
| **卷王** | 脊椎 +2px、领带挺立、眼袋 3px（§3 "¬" 钩） | 凝视屏幕（基）/ 咬笔尖 / 瞳孔放大 | 桌左永远一杯冷咖啡、便签满字 | 被表扬时嘴角上扬但**眼睛不动** —— 笑容贴脸上 |
| **摆烂族** | 单肩下垂 4px、靠椅背、手机反光在脸上 | 空洞看天（基）/ 假打字 / 突坐直 | 椅子斜 2px、抽屉小说封面半露 | 被抓到时起身过快 —— **动作与笑容之间有 1 帧空白** |
| **谄媚族** | 蛋形轮廓、下巴前伸 2px、双手胸前合 | "哦哦哦"嘴型（基）/ 鞠躬 / 沉默皱眉 | 名片夹胸口半露、鞋尖高光像素 | Boss 怒时他笑得最大 —— **五官距离拉宽** |
| **新人** | 窄长无突出、脖子略长、双手夹膝间（站时） | 茫然（基）/ 猛记笔记 / 快哭 | 背包未放柜、工牌绳打结 | 第一次听"加班"时嘴角上扬 0.5 秒后**垮掉** |
| **老油条同行** | 似玩家，茶杯换保温杯（圆柱 2px 粗） | 会意眨眼（基）/ 叹气 / 苦笑 | 桌下拖鞋、文件夹橙色（玩家是灰） | 见新人加班时点头微笑 ——**眼角阴影** |
| **清洁阿姨** | 矮宽（+4 -6px）、拖把作第三支腿 | 面无表情擦桌（基）/ 嘀咕 / 意味深长笑 | 围裙手机屏家庭像素画 | 见人晕倒**继续擦地 —— 嘴角上扬不停步** |
| **Boss** | 剪影顶 +8px（强制感）、双手背后、领带最宽 | 俯视（基）/ 假笑点头 / 沉默皱眉 | 领带夹 logo 像素徽章、皮鞋高光描边 | 被上级批评后转头夸下属 ——**笑容残影留 2 帧** |
| **HR** | 平肩正坐、文件夹永腋下、眼镜**空框无镜片** | 标准微笑（基）/ 翻文件 / 看手表 | 桌上绿植（唯一有生命道具）、"人才"马克杯 | 通知裁员时措辞正向 ——**嘴型与台词对不上** |
| **隔壁部门代表** | 色系蓝 vs 本部门灰、徽章明显、走路快 2 帧 | 微笑递名片（基）/ 装不认识 / 争抢怒目 | 水杯部门贴纸、手机壳同色 | 合作成功时笑对镜头 —— **但身体朝向反方向** |

### 5.3 表情 / 姿态风格

- **帧数上限：3 帧循环**（怠倦→微变→复位）；事件 CG 允许 4 帧单次；全游戏不超 4 帧
- **夸张程度：1.5× 夸张**（与 §3 姿态一致），瞳孔 ±2px、嘴型 ±3px；**喜丧状态允许局部突破到 2.5×**
- **MVP 必备姿态库**（原型共用基底，个别特化）：

| 姿态 | 帧数 | 说明 |
|------|------|------|
| 坐·默认 | 3 | 工位基础 |
| 坐·打字 | 2 | 肩膀循环 |
| 坐·看手机 | 2 | 复用打字手型 |
| 站·正视 | 1 | 对话站立 |
| 站·背对 | 1 | 离开工位 |
| 走·侧面 | 4 | 过道移动 |
| 被骂 | 3 | 肩膀内缩→复位 |
| 喜丧触发 | 4 | 原型专属，**不共用** |

### 5.4 LOD 与展示距离哲学

**主视角**：**侧视角 2.5D** —— 工位横向纵深排列，摄像机固定，玩家可看 2-3 排工位

**基础 Sprite 尺寸**：**32×48 px** —— 在 1920×1080 ×4 放大后占屏 128×192 px，工位格刚好匹配。**超过此尺寸单人产量不可控**。

**LOD 三层**：

| 层级 | 触发条件 | 像素规格 | 细节保留 |
|------|---------|---------|--------|
| 工位远景 | 全局视图 | 32×48 | 仅剪影特征 + 颜色组 |
| 互动特写 | 鼠标悬停 / 选中 | 64×96（×2 放大） | 表情 + 标志物细节 |
| 事件 CG | 剧情触发 | 128×192（独立绘制） | 全细节 + 喜丧状态表情 |

### 5.5 Pixel 产量控制（单人 3 个月生死线）

**每 NPC MVP Sprite 上限：16 张**

| 类型 | 数量 |
|------|------|
| 坐姿（3 帧 × 2 情绪） | 6 |
| 站姿（正 / 背） | 2 |
| 走路（4 帧侧面） | 4 |
| 喜丧触发（专属 2 帧） | 2 |
| 事件 CG（1 张） | 1 |
| 互动特写头像 | 1 |

**总计：16 × 9 NPC + 玩家 20 = MVP 上限 ≤ 164 张**

**复用策略**：
- **上半身帧**（打字 / 看手机）在坐姿原型上换手部即可复用，节省 30% 产量
- **皮肤 palette swap**：所有人类共用同一张皮肤 palette，换色即换种族 / 肤色
- **下半身帧**（腿部 16 px 走路循环）所有人类共用，上半身独立绘制

**绝对不能省**：
1. **剪影辨识特征帧**（茶杯凸起、领带挺立、单肩低垂 等）—— 省了等于原型消失
2. **喜丧触发帧** —— 游戏核心美学载体，每 NPC 至少 2 帧专属
3. **互动特写头像** —— 玩家对话时全程盯着，必须高质量

## 6. Environment Design Language（环境设计语言）

### 6.1 建筑风格 + 办公室类型

**主类型：故意杂糅（互联网大厂为核心 + 老牌私企遗产）**

理由：让每个中国打工人都"认出这里"。单一公司类型产生排他感（"我们公司不这样"），**杂糅让玩家在不同角落各自认领** —— 格子间高密度是大厂、墙上 2016 年 KPI 荣誉榜是老私企、走道共享单车头盔是创业公司遗产。

**建筑年代暗示：2005 年老写字楼 + 表面翻新**
- 外墙：灰色瓷砖（`#8B9EB7`），竖向污渍纹路
- 吊顶：**白炽灯管（不是筒灯）**，部分灯管轻微闪烁（overtime 变体下频率加快）
- 地板：灰白方格瓷砖 —— **地板砖 8×8 px 格，完全对齐工位网格**（强化"格子压人"）

**"第一眼认同 / 第二眼刺破"服务**：第一眼是普通国产写字楼；第二眼发现**电梯按钮只有 B1 和 18 层磨损**（食堂与老板楼），其他楼层几乎完好。

### 6.2 质感哲学

**选择：Clean Pixel 为主 + 局部 2bit Dithering 点缀**

| 流派 | 代表 | 为何本作不主用 |
|------|------|--------------|
| 绘画式（Painted） | Hyper Light Drifter | 单人产量不可控 |
| **网格式（Clean Pixel）** | Stardew Valley | **主选 —— 道具信息密度可读** |
| 喷涂式（Dithered） | SFC FF 系列 | 小分辨率下噪点干扰读取 |

**overtime 专属视觉语法**：overtime 变体阴影过渡、屏幕蓝光晕染使用 **2bit dithering，仅出现在光源边缘 4 px 内**；日间状态不用 dithering。**overtime 在视觉语法上属于"另一种物理定律"**。

### 6.3 道具密度规则

**工位桌面必备清单**（按概率权重）：一次性纸杯 / 外卖袋 / 文件堆 / 工牌背面贴纸 / 绿萝（死的）/ 廉价台历

**工位密度规则**：
- 每工位格（16×16 px 桌面区）：**最低 3 道具，最高 5 道具** *(由 Section 8T.7 C-01 回溯约束 —— 原为 7，改为 5 以保 batching 效率与 <100 draw call 预算)*
- 道具尺寸锁定：小道具 4×4 px，中道具 8×8 px，大道具 8×16 px（文件堆）
- 屏外工位格子应用 culling，不渲染非当前镜头范围内的道具

**各区域密度**：

| 区域 | 密度 | 底线道具 |
|------|------|---------|
| 工位区 | 拥挤 | 纸杯 + 文件堆（缺一不可） |
| 走道 | 稀疏 | 一个被遗忘的快递盒 |
| 茶水间 | 中等 | 饮水机 + 至少 2 个脏杯 |
| 老板办公室 | 空旷 | 1 盆**假**绿植 + 1 个奖杯 |
| 会议室 | 中等 | 翻页日历（永远停在错误日期） |

**个人化道具**（NPC 专属 —— 低成本人物刻画）：
- 卷王：工位只有充电线和水，桌面最干净
- 摸鱼族：手机支架 + 零食袋压文件
- 老员工：泛黄工牌挂件 + 2016 年马克杯
- 实习生：新笔记本（空白页仍可见）

### 6.4 环境叙事导引（5 条核心规则）

1. **"卷王的桌子比其他桌更干净，因为他一直在演"** —— 工位整洁度 ≠ 工作量。一眼可见但需第二眼才读懂反讽。
2. **"老板办公室的绿植是假的"** —— 用 4×4 px 假叶纹理（无渐变纯平色块）区别真绿萝。需走进办公室才识别。
3. **"茶水间饮水机旁'本周换水记录'签名全是同一个人"** —— 文字道具，overtime 状态下字体由灰变红。
4. **"走道的快递盒有脚印印"** —— 快递盒 sprite 有 clean / stepped 两个 variant，按当日 NPC 流量自动切换。
5. **"会议室白板上的方案从未被擦掉"** —— 白板 sprite 叠加历史笔迹 alpha 层，每次会议事件后新增一笔，越旧越淡。

**发现成本分级**：
- **一眼可见**（叙事锚点）：道具密度 / 区域灯光 / 人流方向
- **二次进入才注意**（刺破彩蛋）：饮水机签名 / 白板笔迹 / 卷王桌空旷之谜

### 6.5 时间流逝的视觉呈现

**日内变化**（扩展 Section 2 _day / _overtime）：
- **上午 9 点**：白炽白主导，屏幕蓝光点缀
- **下午 3 点（_day 变体）**：打工人黄加热 +10% 亮度 —— 表现午后昏睡感
- **overtime**：屏幕蓝光覆盖率 +40%，白炽灯局部关灯产生黑斑

**累积性变化（不重置）—— 所有变化均为可存档 sprite state**，通过 Godot 4.6 `@export` 变量绑定 variant index：

| 时间单位 | 环境变化 | 存档字段 |
|---------|---------|---------|
| **每周** | 玩家桌面多 1 个水印杯底 sprite | `desk_stain_count: int` |
| **每月** | 走廊通知栏新增 1 张过期通知（旧通知褪至 `#BBBBBB`） | `notice_board_age[]` |
| **季度** | 茶水间墙面出现 1 条新裂缝（4 px 灰线） | `break_room_cracks: int` |
| **周年日** | 横幅"奋斗 X 周年"覆盖老板办公室，overtime 下变咖啡渍棕黑 | `anniversary_year: int` |

## 7. UI/HUD Visual Direction（UI 视觉方向）

*本节由 art-director + ux-designer 并发起草并对齐。Lean mode 下完成 3 项关键张力的创意决策：UI 警告改色盲安全黄 / MVP 做齐全 diegetic / KPI 动画全程可跳过 + 精缩版。*

### 7.1 Diegetic vs Screen-Space 分工

**原则：感知 → diegetic，系统 → screen-space。**

| 元素 | 归属 | 理由 |
|---|---|---|
| 精力 / 摸鱼槽 | **Diegetic**（玩家屏幕像素条） | 工位即战场 |
| 任务清单 | **Diegetic**（桌角便利贴） | 强化"活在格子间"幻觉 |
| Boss 心情 | **Diegetic**（走廊情绪板） | Pillar #2 叙事即机制 |
| 存档 / 暂停菜单 | Screen-space | 系统级，不属于世界 |
| 警告弹窗 | Screen-space | 需要打破第四面墙 |
| 月末 KPI 考核 | Screen-space + 可跳过（见 7.4） | 强制演出有剪裁 |

混合时 screen-space 用 `#1A2A38` multiply 压世界底层，明确层级。

**MVP 决策**：**全面做齐 diegetic UI**，不走 screen-space fallback。gamepad 兼容通过焦点环跳转链（见 7.5）解决。

### 7.2 字体层级

| 用途 | 字体 | 字号 | 补充规则 |
|---|---|---|---|
| 标题 | 方正公文宋 | 16 px | 硬边衬线，公文感 |
| 正文 / 对话 | 思源黑体 **Regular** | 12 px | **最低对比度 WCAG AA 4.5:1**；白炽白背景下保证笔画可读 |
| 数字 / 数值框 | 5×7 bitmap 等宽 | 8 px | **整数像素对齐（no sub-pixel）**，UI 程序员须注明 |
| 系统提示 / Boss 台词 | 站酷快乐体 | 14 px | 与警告黄 `#F5C400` 配对；**必加 ! 符号冗余语义** |

**禁用**：10 px（笔画粘连）。**最小字号底线 8 px**。
**英文版预留**：30% 文本膨胀空间，超出时降 11 px 或启用文本自动缩减。
**反讽钩（Pillar #4 承载点）**：死亡结算画面标题用**站酷快乐体写"恭喜晋升"** —— 字体的欢脱 + 内容死亡 = 喜丧美学刺痛感。

### 7.3 图标：像素图章（Pixel Stamp）风格

**尺寸层级**：16×16 功能 / 24×24 状态 / 32×32 卡片封面

**理由**：与 SFC 美学一致；2-3 色限色逼出会意图形；"盖章"语感强化复印机 UI 形状语法。

**色盲双重编码铁则**：所有状态图标必须形状 + 颜色双重编码 —— KPI 达标绿 `↑↑` 箭头 + 纯色 / 警告黄 `!` 感叹号 + 1 px 斜线纹理；**不能让颜色独立传达危险等级**。

**黑色幽默图标示例**：

| 图标 | 设计 |
|---|---|
| **摸鱼** | 16×16，向上游动小鱼像素，但鱼身是指针光标 —— 鱼即鼠标，摸鱼即摸鼠标（双关） |
| **装病** | 24×24，温度计插戴眼镜的骷髅头，读数 999°；图章边锯齿如医院盖章痕迹 |
| **加班** | 32×32，钟表指针走完一圈又多半圈，时针分针交叉成 X —— 时间已"死"；配色反转为 `#2A1F14` 底 `#E8E0CC` 针（黑白颠倒） |

### 7.4 UI 动画 feel：**官僚式迟钝严格限于叙事演出层**

| 场景 | 哲学 | 规格 |
|---|---|---|
| **按钮 hover / 焦点切换 / confirm** | **即时响应 ≤ 1 帧** | 0 缓动 —— 操作反馈绝不"迟钝" |
| 面板弹出（叙事演出） | 官僚式慢 | 3 帧 @ 8 px/帧 |
| 数值变化（掉血 / 加状态） | 逐格填色 | 闪警告黄 1 帧 → 目标色 1 帧 |
| 卡片抽起 | 喜丧式夸张 | 2 帧 overshooting +4 px → 归位 |
| "领导出现"警告弹窗 | 打印机出纸压迫 | 5 帧扫描线逐行显示 |

**✨ 跳过铁则**：
- 所有 ≥ 2 秒演出动画必须支持**任意键跳过**
- **月末 KPI 考核打印动画**：全程可跳过；**额外提供"精缩版"快动画**（1/3 时长，同节奏感），玩家可在设置里切换为默认精缩模式
- 操作反馈（按钮、焦点、输入 confirm）永远 ≤ 1 帧即时响应 —— Frenetic 决策时刻不能有输入堵塞感

### 7.5 Gamepad / Focus 态（Switch 预留）

- **Gamepad focus 态视觉**：外框高亮 `#C8963C` + 2 px 跳动偏移（区别于 hover 盖章双线）
- **Diegetic UI 的 D-Pad 导航**：为每个 diegetic 元素（便利贴、桌角屏幕、情绪板…）定义 **Tab-order 映射到 D-Pad 方向**，形成焦点环跳转链。`art-bible` 的 UI 实现阶段，每个 diegetic 元素的 UI 程序员需标注逻辑 Tab 位次。
- **Switch 阶段回退选项**：若焦点环实现复杂度过高，可在 Switch port 时切换到 screen-space 摘要面板 —— 但此决策**推迟到 Scope Tier 野心版阶段**，MVP 不做 screen-space fallback。

### 7.6 自动存档 + 地铁 5 秒进入（Pillar #5）

- **自动存档时机**：每个 AP 消耗后自动保存
- **主菜单默认焦点**：**"继续游戏"**（非"新游戏"）
- **主菜单 → 继续游戏路径 ≤ 2 次交互**
- 地铁 5 分钟场景下一局应可完成：打开 → 继续 → 玩 2-3 天 → 自动存档后直接关闭 app

## 8. Asset Standards（资产标准）

*本节由 art-director（审美 / 命名 / LOD 规格）+ technical-artist（引擎硬约束 / batching / shader）并发起草后合并。冲突已解决：道具上限 7→5（回溯 §6.3）、_day/_overtime 用 CanvasModulate、CG 硬上限 128×192、dithering 仅 overtime、variant 优先 atlas frame、Shader Baker 启用。*

### 8.1 文件格式 + 命名约定

**文件格式**：
- Sprite：`.png`（RGBA 8-bit，强制 indexed PNG-8 导出保 alpha）
- Palette 源：Aseprite `.aseprite`（含内嵌调色板）+ 导出 `.gpl`（GIMP 兼容）
- `.pal` 不推荐（跨工具互通性差）

**命名规则**（字段顺序固定）：
```
[类别]_[对象]_[状态/变体]_[帧编号或尺寸标记].[ext]
```

例：
- `npc_bosslady_idle_day_01.png`
- `npc_player_walk_overtime_03.png`
- `env_desk_stain_week4_16x16.png`
- `ui_btn_primary_normal_48x16.png`
- `vfx_sweat_loop_16x16_01.png`

**目录结构**：
```
assets/
├── sprites/
│   ├── npc/
│   │   ├── atlas/          # npc_atlas_office_a.png 等（按场景分包）
│   │   └── frames/         # SpriteFrames .tres 资源
│   ├── env/
│   │   ├── props/          # 道具
│   │   └── variants/       # 累积环境 state sprites
│   ├── ui/                 # ui_atlas.png（单张 2048×2048）
│   └── cg/                 # CG 独立文件（不进 atlas）
├── shaders/
│   ├── palette_swap.gdshader
│   └── dither_overlay.gdshader
├── palettes/               # .aseprite 源板 + .gpl 导出 + 8×1 PNG LUT
└── source/                 # .aseprite 源文件（Git LFS）
```

### 8.2 像素尺寸层级

| 类别 | 尺寸层级 | 说明 |
|---|---|---|
| NPC / 玩家 | 32×48 / 64×96 / 128×192 | LOD 0/1/2 |
| 道具（桌面） | 4×4 / 8×8 / 8×16 | 对齐 16×16 工位格 |
| 地板砖 | 8×8 | 2×2 拼一工位格 |
| 工位模块 | 16×16（单格）/ 32×16（双格桌） | 严格正交 |
| 房间模块 | 以 16×16 为基础单元拼接 | 不设固定上限 |
| UI 图标 | 16×16 / 24×24 / 32×32 | |
| UI 按钮 | 48×16（标）/ 96×16（宽） | 0 圆角 1px 边框 |
| UI 对话框 | 128×32（单行）/ 128×64（多行） | 内边距 4px |
| **事件 CG** | **128×192 硬上限** | 超过此尺寸需 tech-director 审批（防显存爆掉）|
| VFX sprite | 8×8 / 16×16 / 32×32 | 按特效体积分三档 |

### 8.3 Godot 4.6 导入配置

**TextureImporter 参数**（全局 sprite 默认）：

| 参数 | 值 | 原因 |
|---|---|---|
| `filter` | `Nearest` | 保持像素锐利 |
| `mipmaps/generate` | `false` | 2D 像素不需要 LOD 模糊，省 33% 显存 |
| `fix_alpha_border` | `true` | 防半透明边缘颜色渗漏 |
| `compress/mode` | `Lossless` | 不破坏 indexed 调色板 |
| `detect_3d` | `false` | 强制 2D 路径 |
| `normal_map` | off | SFC 无法线贴图需求 |
| `sRGB` | 开启（Color 纹理）/ 关闭（palette LUT） | LUT 必须 linear |

**`project.godot` 全局**：
```ini
[rendering]
renderer/rendering_method="forward_plus"
textures/canvas_textures/default_texture_filter=0  # Nearest
textures/default_filters/use_nearest_mipmap_filter=false
anti_aliasing/quality/screen_space_aa=0  # SMAA 关闭，保像素锐
shader_compiler/bake_ubershader=true  # Shader Baker（4.5+）开启
```

### 8.4 Palette 管理

**生产约束：每 sprite ≤ 16 色（含透明索引 0）**

调色板三主板：
| 板名 | 色槽数 | 用途 |
|---|---|---|
| `pal_skin_base.gpl` | 8 色 | NPC 皮肤共用基础 |
| `pal_world_office.gpl` | 16 色 | 场景/道具/背景 |
| `pal_ui_clean.gpl` | 12 色 | UI Clean Pixel 主调 |

**overtime 皮肤换色**：使用 **shader palette swap** 实现（不新增 sprite 文件）
- `pal_skin_overtime.gpl` 定义 8→8 映射，导出为 **8×1 px PNG LUT**
- Shader 用 R 通道作 index，采样 LUT 得目标色
- 运行时仅改 `palette_index` uniform，零开销
- 累积环境 state（脏桌面、墙裂缝）用独立 sprite 图层叠加，不用 palette swap

### 8.5 Rendering / Batching 硬约束

**同屏 sprite 软上限 200 个**；超出后 batching 效率下降。

**典型"白天行动"场景 draw call 预估**：
| 元素 | 数量 | draw calls |
|---|---|---|
| NPC（同 atlas） | 8-12 | 1-2 |
| 道具（按区 atlas） | ≤ 5/格 × 9 格 ≈ 45 | 2-4 |
| 背景 TileMapLayer | 2-3 层 | 2-3 |
| UI CanvasLayer | HUD + 通知 | 2-4 |
| CanvasModulate 色调 | 1 | 0 |
| **合计** | — | **~10-15** |

远低于 100 draw call 预算。

**关键批处理约束**：Z 序中断、不同材质、不同 shader uniform 会打断 batch。美术规划 Z 序时需与 UI 程序员对齐。

**NPC Atlas 策略**：按**场景**分 atlas（`npc_atlas_[scene_id].png`），当前场景外 NPC atlas **不加载**。**同屏 NPC 软上限 12 个**。

> **预估显存预算**：NPC 按场景分 atlas 后常驻 ≈ 10-12 MB；UI atlas ≈ 16 MB；环境 ≈ 2-4 MB；palette LUT < 0.1 MB。**MVP 运行时总纹理内存 ≤ 40 MB**（远低于 500 MB 内存上限）。

### 8.6 渲染哲学：零 Light2D + CanvasModulate 色调

**本作不使用 Light2D**。_day / _overtime / KPI 等色调切换全部通过 `CanvasModulate.color` + `Tween` 0.3 秒插值完成。理由：

- `CanvasModulate` 零额外 draw call，与 SFC 固定色调风格完美匹配
- Light2D 在 Forward+ 2D 下每盏会让所有受光 sprite 重绘一次，4 盏 × 210 sprite = 840 次额外重绘
- 像素美术不需要 normal map 立体感，Light2D 收益不抵成本
- `self_modulate` 用于单 NPC 高亮（overtime 状态个人色调）
- **如未来 Scope 野心版要做 Light2D 局部氛围光**：硬上限 1 盏（无阴影），仅限关键剧情演出

### 8.7 动画系统约定

| 动画需求 | 推荐节点 | 原因 |
|---|---|---|
| NPC 3 帧循环（走/待机） | `AnimatedSprite2D` + `SpriteFrames` | 编辑器友好，产量优先 |
| NPC 复杂状态机（未来） | `AnimationPlayer` + `AnimationTree` | 需 blend 时升级 |
| UI 按钮 hover | `theme` 属性直接切换 | 像素风无需 Tween |
| UI 面板弹出（3 帧） | `Tween` 代码创建，用完销毁 | 避免常驻 AnimationPlayer |
| 累积环境 state 切换 | `AtlasTexture.frame_coords` 信号驱动 | 同 atlas 零额外内存 |

### 8.8 Shader 管理

- **palette swap shader**：1 个 ubershader，`palette_index` uniform 区分变体
- **dither overlay shader**：全屏 CanvasLayer + ColorRect，**仅 overtime 激活**（非 overtime `visible=false`）
- **运行时活跃 shader 种类 ≤ 10**
- **Shader Baker（4.5+）启用** —— 首次进场景无 hitching

### 8.9 产量质检清单（Per Sprite）

1. **剪影辨识**：灰度模式下轮廓可读
2. **调色板超限**：Aseprite 确认索引色 ≤ 16
3. **像素对齐**：画布原点对齐网格，无亚像素偏移
4. **命名合规**：符合 8.1 规则
5. **Godot 导入预览**：Inspector 确认 filter=Nearest 无模糊

### 8.10 资产验证流水线

**建议实现 `tools/asset_lint.gd`（编辑器 Tool 脚本）**，检查：
1. `assets/sprites/` 下 PNG 是否 `import_preset = "2D Pixel"`
2. NPC sprite 尺寸符合 32×48 / 64×96 / 128×192 约定
3. Atlas 单张 ≤ 2048×2048
4. SpriteFrames 命名符合 `npc_[id]_[state].tres`
5. Shader 文件 palette LUT 绑定检查（孤立 uniform 告警）

### 8.11 版本控制

- **`.aseprite` 源文件跟随 git**（`assets/source/`），用 **Git LFS** 管理
- 预估源文件量：164 张 NPC × 平均 40 KB ≈ 6.5 MB，可接受
- `.png` 导出文件同样入库作为引擎直接引用的交付产物
- 交付单位：每 NPC 全部动画帧打一个 PR；场景元素按区域打包

### 9.2 每条参考的使用规则

**R1 — Chrono Trigger（像素精度）**
角色任何部位的高光面积**不超过该部位总像素数的 20%**；打光方向全局统一**左上 45°**；overtime 场景可叠加屏幕正前方补光，但补光色**必须是屏幕蓝光 `#2C4A6E`**，不用白光。

**R2 — 《生活万岁》（环境真实感）**
绘制格子间背景时，远景隔断饱和度 ≤ **前景 40%**，以此制造纵深感**替代透视线**的压迫效果。

**R3 — 《中国式家长》（剪影测试）**
每个新 NPC 完成后必过"剪影测试"：导出为纯黑剪影，在 **16×32 px 缩略图尺寸下职级必须可辨**（主管有肩章轮廓或领带宽度差异）。

**R4 — Papers, Please（UI 排版）**
所有 UI 数值框遵循"格线即边距"：**边框粗细 = 内部网格线宽（2px）**；不加阴影/圆角；交互状态的区分通过**底色明度差 ±15%**，不改变边框颜色。

**R5 — 朱德庸（喜丧锚点）**
每个场景必测"视觉重心"：场景中必须有**一个元素像素面积 ≥ 同层级物件 1.5×**，承担"荒诞锚点"功能；该元素使用**点缀色**（老板金 / 俗艳粉 / 片尾曲红），其余环境保持主调色板。
