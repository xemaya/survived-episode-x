# Game Concept: 《活过第 X 集》 (Survived Episode X)

*Created: 2026-04-22*
*Status: Draft*
*Engine: Godot 4.6 (推荐) | Platform: PC / Steam*

---

## Elevator Pitch

> 一款像素风的**反向 KPI 办公室生存模拟游戏**。你扮演一个中国职场老油条，每天 8 个行动点要在"被开除"和"太优秀下关被涨 KPI"之间走钢丝 —— 不能太努力（优秀的回报是更高的 KPI）、不能太摆烂（绩效不达标直接 game over）、也不能太累（精力也是资源）。你的目标不是升职加薪，是**活过多少集**。
>
> 像《大多数》遇到《死亡与税收》，**AND ALSO 胜利条件是"维持平庸"**。

## Core Fantasy

**"我看穿了这家公司，但我不反抗 —— 我只是尽量活得久一点、久到让同事都记不清我来了几年。"**

这是现代中国打工人的集体无意识幻想 —— 不是"逆袭成为 CEO"，而是"在结构性荒诞里用小聪明换取小体面"。玩家扮演的老油条不是英雄也不是失败者，是一个**清醒的共谋者**：他知道 KPI 是骗局、知道 PUA 话术的套路、知道周年庆是笑话，但他仍然每天 9 点打卡、5 点半溜走 —— 因为工资真的需要，因为妈还等他养老，因为除此之外的选择更糟。

这款游戏给玩家提供的，是**把这种日常的无力感和小聪明变成可玩、可笑、可交互的博弈**。

---

## Unique Hook

**"维持平庸是一种技术活。"**

像《大多数》一样讲述中国底层生活的真实感，**AND ALSO** 把胜利条件反转 —— 传统模拟游戏奖励玩家变强，这款游戏的核心博弈是**如何维持一个"不够优秀到被涨 KPI、不够差到被开除、不够累到透支精力"的三维安全区**。

具体机制实现：
- 每天 8 AP，加班拿物质但**推高下关 KPI 阈值**（甜头即毒药）
- 月末 KPI 达标后系统按"努力 + 潜力 + 工龄"三维度惩罚性涨基准
- 工龄是隐性倒计时 —— HR 会系统性给老员工压不合理目标，让他自动滚蛋

这不是表面讽刺，**这是用游戏机制把"职场 PUA 结构"建模成了玩家每天都要对抗的系统**。目前市面没有其他游戏用这种机制把主题内嵌到系统里。

## Player Experience Analysis (MDA Framework)

### Target Aesthetics（玩家的情绪体验）

| Aesthetic | Priority | How We Deliver It |
| ---- | ---- | ---- |
| **Discovery**（发现） | **1**（主要） | 每局解锁新 NPC/事件/卡；剧情暗线；看穿系统的 aha 时刻 |
| **Narrative**（叙事） | **2**（主要） | 每张行动卡 = 小剧本；8-10 NPC 有人物弧；黑色幽默 tone |
| **Challenge**（挑战） | **3** | 反向 KPI 博弈、围棋式布局感；挑战是"活多久" |
| **Submission**（放松/仪式） | **4** | 每日打卡的节奏感；5 分钟地铁可玩；不烫手 |
| **Fantasy**（代入） | **5** | 扮演清醒的老油条，一个特定文化下的集体原型 |
| **Sensation**（感官愉悦） | 6 | 像素美术 + 环境音，支撑但不是卖点 |
| **Expression**（表达） | 7 | 有但不核心 —— 个人化"摸鱼风格" |
| **Fellowship**（社交） | N/A | 单机内省游戏，Anti-pillar 明确排除 |

### Key Dynamics（涌现玩家行为）

我们希望玩家自然开始做以下事：
- **主动维持平庸**：拒绝突出表现的机会，刻意"差不多就行"
- **策略性演技**：学会在合适时机"装忙"/"装累"/"装请假"
- **NPC 情感投资**：关心 Lisa 是不是真要跳槽、清洁阿姨的儿子考上学了没
- **看穿系统**：在周报/月报/季报的数据表层下识别老板的心理模型
- **社区共创段子**：把游戏里的经典事件截图发朋友圈/小红书（天然营销）

### Core Mechanics（3-5 个核心系统）

1. **AP 经济系统**：每日 8 点，早退奖励精力、加班奖励物质但惩罚长期
2. **行动卡 + 微叙事**：每张行动卡触发小剧本场景 + 数值/关系/flag 反馈
3. **三层 KPI 结算**：周度（周报呈现）/ 月度（通关考核）/ 季度（大考核）
4. **NPC 关系网**：8-10 个有刻画的 NPC，关系变化改变事件触发概率和剧本分支
5. **反向涨阈值机制**：月末 KPI 达标按努力/潜力/工龄三维度惩罚性提高下月基准
6. **跨局内容解锁**（完整版）：每局结束解锁新 NPC/事件/卡/结局，博物馆式进度

## Player Motivation Profile

### Primary Psychological Needs Served (SDT)

| Need | How This Game Satisfies It | Strength |
| ---- | ---- | ---- |
| **Autonomy** | 每天 8 AP 怎么花、早退还是加班、和哪个同事结盟 —— 玩家做主 | **Core** |
| **Competence** | 渐进"看穿系统"的 mastery：识别工龄惩罚、解锁剧情分支、优化反向 KPI 策略 | **Core** |
| **Relatedness** | 单机但 NPC 关系浓厚 —— Lisa、老板、清洁阿姨都是有血有肉的人物 | **Core**（最关键差异化维度）|

### Player Type Appeal (Bartle Taxonomy)

- [x] **Explorers** —— 解锁内容 + 发现隐藏机制 + 看穿系统的 aha 时刻
- [x] **Socializers** —— 不是多人社交，是单机 NPC 关系网（Disco Elysium / 大多数 式）
- [ ] **Achievers** —— 有"活过多少集"的分数但不是主要驱动
- [ ] **Killers/Competitors** —— 明确不服务此类玩家

### Flow State Design

- **Onboarding curve**: 前 3 天（游戏内）隐形引导，用预置事件教 AP 系统；第 1 次月末 KPI 结算用老同事点评解释"涨阈值"机制，避免玩家觉得被坑
- **Difficulty scaling**: 难度由三维度惩罚曲线自动调整 —— 越优秀越难、工龄越长越难；玩家感受是"职场天花板逐步压下来"
- **Feedback clarity**: 每日总结明示数值变化；周报呈现用"本周有趣事件"+数值快照；月末 KPI 结算用戏谑化结果报告
- **Recovery from failure**: Game Over 不可逆（单局），但跨局解锁机制让下一局开始更有趣 —— 失败是剧终不是惩罚

## Core Loop

### Moment-to-Moment（30 秒 / 1 个 AP）

选一张行动卡 → 弹出小剧本场景（3-5 秒文字+像素动画）→ 数值反馈（KPI / 精力 / 关系 / 隐藏 flag）→ 回到行动卡界面。

**这个 30 秒必须本身就好玩** —— 即使没有长线进度，读一段小剧本 + 看数值反馈已经构成一个完整的微闭环。

### Short-Term（一天 ≈ 2-3 分钟）

1. **早晨事件预告** —— 1 张图 + 2 行字（"老板今天出差"/"Lisa 项目 deadline"）
2. **白天行动阶段** —— 用 8 AP，每张卡一个剧本 + 反馈
3. **下班决策** —— 早退（AP 有剩）/ 准点 / 加班（超 AP）
4. **今日总结** —— 数值变化 + 关键事件回顾 + 明日预告

### Session-Level（10-15 分钟 / 一周）

- 周一到周五（5 天打卡）→ **周五：周报呈现**（本周干了啥有意思的事 + 数值快照）→ **周末两天**（躺平回精力 / 社交触发支线 / 跳过）

地铁套：5 分钟 = 大约 2 天；15 分钟 = 整整一周。玩家可随时存档退出（任意行动点后）。

### Long-Term Progression（一局 / 一次入职 → 开除）

- 一个月 = 一关（4 周）；每月末 KPI 考核，未达标 GAME OVER
- 每年 12 个月 → 周年庆事件（《破事精英》梗）
- **一局必会走向 GAME OVER** —— 工龄惩罚让最终失守是数学上的必然
- Game Over 结算画面：**"你的《职场》连续剧演完第 X 集，评分 X.X 分 —— 感谢观看"**

### 跨局循环（多周目 / Meta 进度）

- 每局解锁：新 NPC、新事件、新行动卡、新结局
- 博物馆式收集：解锁全部内容
- **不解锁"角色变强"** —— 违背 Pillar 1

### Retention Hooks

- **Curiosity**: 未解锁的 NPC/事件/结局；某条剧情线还没走到
- **Investment**: 上局认识的 Lisa 在新一局会有不同故事线吗？
- **Mastery**: 能不能活到第 24 集（2 年）？能不能解锁"退休"结局？
- **Social**: 朋友圈/小红书分享典型事件截图（用户共创营销）

## Game Pillars

### Pillar 1: 平庸是一种艺术

游戏奖励"维持平庸"，惩罚任何偏离舒适区的表现。奖励曲线必须是倒 U 型。

*Design test*: 如果一个新功能奖励玩家"变得更优秀/更强/更努力"，重新设计它 —— 让它有甜头也有毒。加班的物质回报必须绑定下关 KPI 涨幅。

### Pillar 2: 叙事即机制

每个数值变化都包裹在小剧本里，没有孤立的数字。NPC 是有血有肉的，不是数据点。

*Design test*: 设计一张新行动卡之前，先写那张卡的剧本。没剧本就没卡。任何 UI 反馈都要有文本同步（"Lisa 关系 -2" 必须伴随一句她的对白或动作描述）。

### Pillar 3: 死亡是注定的

GAME OVER 不是失败，是剧终。玩家玩的是"活多长"，不是"能不能赢"。永远不会有"游戏通关"。

*Design test*: 永远不要让玩家以为能赢到底。月末 KPI 达标必须让下月更难，不是更简单。任何让玩家"稳赢"的路径都是 bug，不是 feature。

### Pillar 4: 苦中作乐（黑色幽默优先）

基调是黑色幽默 —— 失败是段子、PUA 话术被解构、结构性痛苦被调侃但不说教。

*Design test*: 所有文本先过"朋友圈测试" —— 一个真正的职场人会不会转发这段话？不会就重写。不要煽情，不要正能量，不要励志。

### Pillar 5: 地铁可玩性

5 秒进入、5 秒暂停 —— 游戏服从玩家的时间表。

*Design test*: 任何新功能都要过"地铁测试" —— 能在地铁上玩 5 分钟然后放下吗？不能就砍掉或重新设计。存档必须在任意行动点后都可行。

### Anti-Pillars（我们绝不做的事）

- **NOT 升职打怪**：拒绝角色数值变强曲线。违背 Pillar 1 ——角色变强意味着玩家预期能战胜 KPI。
- **NOT 励志叙事**：拒绝"努力就有回报"的剧情线。违背 Pillar 3 ——主题是结构性荒诞，不是个人奋斗的小宇宙。
- **NOT 反应/动作类操作**：没有 QTE、节奏、打击感。违背 Pillar 5 + 围棋布局感的核心体验。
- **NOT 多人/联机/真人排行榜**：分散"孤独老油条"氛围聚焦，小团队做不起来，分散主题注意力。
- **NOT 付费解锁内容**：所有内容通过游戏内解锁。讽刺游戏里搞微交易 = 讽刺不成反被讽刺。

## Risks and Open Questions

### Design Risks

- **反向 KPI 的新手教学**（MED-HIGH）—— 玩家第一次可能不理解"优秀会让下关更难"，感觉被游戏坑。**缓解**：前 2-3 天隐形引导 + 第一次月末结算用老 NPC 点评"哎看着吧，下月你就惨了"
- **叙事 + 节奏平衡**（MED）—— 每张卡弹剧情可能让"地铁可玩性"失守。**缓解**：剧本三档（闪现/长/纯数值），玩家可调叙事密度设定
- **8 AP 系统可能沦为机械分配**（MED）—— 原型必须验证是真决策还是走流程。**缓解**：/prototype 环节重点验证

### Technical Risks

- **事件系统架构**（LOW-MED）—— 数据驱动的事件 + NPC 关系交互设计需要前期架构决定
- **像素美术产出量**（MED）—— 单人做 10+ NPC 立绘 + 事件插画是重担。**缓解**：MVP 只做默认立绘 + 文字表达情绪

### Market Risks

- **审查风险（国内发行）**（MED）—— 黑色幽默 + 职场批判可能踩红线。**缓解**：Steam 国际区无虞；国内需法务审查或准备温和版
- **Steam 关键词竞争**（LOW-MED）—— "模拟经营"赛道拥挤，但"反向 KPI"的差异化 hook 独特

### Scope Risks

- **文本产出量 + 黑色幽默稳定度**（**HIGH**）—— 80-120 个事件剧本对单人是巨大工作量；tone 失稳一段就毁氛围。**缓解**：30-40 关键事件手写，其余模板化；找 1-2 个早期测试员审 tone

### Open Questions（需 /prototype 验证）

1. **8 AP 分配是否有真决策感？** → /prototype 第一阶段验证核心循环
2. **叙事密度 vs 节奏是否能平衡？** → /prototype + playtest 观察玩家中途是否跳文字
3. **第一次月末被涨 KPI 时玩家情绪反应？** → 早期 playtest 重点观察 —— 是笑还是骂，这是生死问题

## MVP Definition

**核心假设**：反向 KPI 循环在 3 个月关卡（游戏内 12 周）内对核心目标玩家是好玩的。

**Required for MVP**:
1. 8 AP 每日系统（含早退/加班的差异化反馈）
2. 每日 → 每周 → 每月 KPI 三层结算（含月末三维度涨阈值机制）
3. 8-10 个有人物刻画的 NPC（各有 3-5 个关键事件）
4. 30-40 张行动卡 + 80-120 个事件剧本（30-40 手写 + 其余模板化）
5. 1 个清晰的 GAME OVER 体验（"活过第 X 集"戏谑结算）
6. 1 个办公室场景（1 层 1 部门）
7. 关卡上限：3 个月（过第 3 个月月末自动触发 demo end，预告完整版）

**Explicitly NOT in MVP**（推迟到完整版）:
- 多部门 / 多办公室
- 周年庆事件（推到 VS）
- 多结局分支（MVP 只 1 种 GAME OVER）
- 跨局 meta 解锁（"内容型成长"）
- 美术事件 CG
- 英文本地化
- 语音

**推荐目标**：先打到 MVP，基于 MVP 反馈决定是否扩到 VS。**不要**一开始就瞄准完整版 —— 第一款 + 几个月 + 完整版 = 必烂尾。

## Visual Identity Anchor

*（占位 —— 由 `/art-bible` 在下一阶段正式展开）*

**方向锚点**：**现代中文办公区、2.5D 像素风**

**风格参考组合**：
- 《破事精英》剧集的视觉调度（饱和但燥郁的色调、日光灯刺眼、工位间的格子密度）
- 《中国式家长》的成人像素质感（不是 8-bit 怀旧，是当代中国生活实感像素）
- 《Papers, Please》的灰度压抑 tone（在关键时刻使用）

**一句话视觉规则**：*"画面要让一个真实的中国白领，瞄第一眼就觉得'这就是我办公室'。"*

**支撑视觉原则（待 /art-bible 确认）**：
1. **空间密度** —— 工位网格密集、物品繁多（水杯、绿萝、加班零食堆），传达"生活被压进方格子"
2. **光线语言** —— 日光灯惨白 + 电脑屏幕蓝光 + 窗外偶尔的日落，传达时间感和精神状态
3. **人物刻画** —— 每个 NPC 有清晰的"办公室类型"视觉标签（谄媚族、摆烂族、卷王、老油条、新人），但在默认立绘内隐含个性细节

**色彩哲学占位**：主色调是**疲倦的黄 + 办公蓝灰 + 咖啡渍棕**，点缀色是**KPI 表格红 + 老板金 + 周年庆俗艳粉**。黑色幽默气质要靠色彩的"熟悉感+讽刺性"传递。
