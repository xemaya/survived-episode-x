### 3.3 .ink 格式速成

如果你不熟悉 Ink，先**完整读一遍 designer 写的 `daily-choices.ink` 3 个 sample stitches**，然后看以下速查表：

| Ink 语法 | markdown 对应 | 含义 |
|---|---|---|
| `// 注释` | `> 备注` | 单行注释 |
| `=== choice_name ===` | `### Choice N · 名 · 类目` | Daily choice stitch |
| `# tag: value` | `**触发**：...` 等 metadata | runtime hint |
| `* [选项 ≤ 4 字]` | `**[A 选项]** —— 后果` | 选项 + indented 后果 |
| `~ var = expression` | `→ 钱 -38 / 状态 +5` | 变量赋值 |
| `~ check_state_after_choice()` | （无）| 调用 helper function（**每 stitch 末必须**） |
| `-> DONE` | （无）| 结束当前 stitch（runtime 决定下一步） |
| `_text_` | `_text_`（一致） | italic 笑天内心独白 |
| `INCLUDE episode-1.ink` | （无）| 引入 VAR 声明 + helper functions |

### 3.4 翻译模板

每个 markdown choice 翻译成 .ink stitch 用以下模板（参考 designer 的 daily-choices.ink）：

**Markdown 输入**：
```markdown
#### Choice 23 · 跟 Lisa 拼奶茶第 12 次 · 午餐

**触发**：12:30。前置 flag = Lisa score ≥ +5
**速度**：标准 (~3 行)
**解锁**：基础

> Lisa 凑过来："拼一杯吗？第二杯半价。"
>
> 你和她已经拼过 11 次。

**[A 拼]** —— 喜茶第二杯半价。两人 38 块。Lisa："今天我请。" `→ 钱 -19 / 状态 +5 / Lisa score +3`
**[B 我减肥呢]** —— 她说"哦"，自己点了。 `→ 钱 0 / 状态 0 / Lisa score -2`

> _拼到第 12 次，奶茶不再是奶茶。_
> _是一个 unspoken contract——她孤独，我也是。_
```

**.ink 输出**：
```ink
// ----------------------------------------------------------------------------
// Choice 23 · 跟 Lisa 拼奶茶第 12 次 · 午餐
// ----------------------------------------------------------------------------

=== choice_lisa_milk_tea_12th ===
# category: lunch
# season_unlock: any
# time_filter: lunch
# weekday_only
# cooldown_episodes: 1
# frequency_per_series: 24
# npc_focus: lisa

Lisa 凑过来："拼一杯吗？第二杯半价。"

你和她已经拼过 11 次。

她从不主动说拼下一次。但每次她都问。

* [拼]
    喜茶第二杯半价。两人 38 块。
    Lisa："今天我请。"——你知道下次会轮到你。
    ~ money = money - 19
    ~ state = state + 5
    ~ lisa_score = lisa_score + 3

* [我减肥呢]
    她说"哦"，自己点了。她的杯子比平时小一号。
    ~ lisa_score = lisa_score - 2

- _拼到第 12 次，奶茶不再是奶茶。_
- _是一个 unspoken contract——她孤独，我也是。_
- _我们都没说出来。_

~ check_state_after_choice()
-> DONE
```

注意翻译要点：
1. **markdown `#### Choice N · 名 · 类目` → `// ---` 注释 banner + `=== choice_短拼音名 ===`**
2. **markdown `**触发**` `**速度**` `**解锁**` → `# tag: value` 块**
3. **markdown `> 文本` → 直接段落（去掉 `>` 前缀）**
4. **markdown `**[A 选项]** —— 后果 → 数值变化` → `* [选项]` + indented 后果 + `~ var = ...`**
5. **markdown 后果分隔 → indented 多行段落**
6. **markdown `→ Lisa +3` → `~ lisa_score = lisa_score + 3`**
7. **markdown 末尾 `> _笑天独白_` → `- _独白_`（共享 fallthrough block）**
8. **每个 stitch 末加 `~ check_state_after_choice()` 然后 `-> DONE`**

**变量名约定**（跟 designer episode-1.ink 顶部声明保持一致）：

| markdown 写法 | .ink 变量名 |
|---|---|
| 钱 | `money` |
| 状态 | `state` |
| KPI | `kpi` |
| Lisa score | `lisa_score` |
| David score | `david_score` |
| 王总监 score | `wang_score` |
| Zoe score | `zoe_score` |
| 李阿姨 score | `li_score`（但李阿姨 score 不影响机制——你写 `~ li_score = li_score + 0` 即可，纯 flavor） |
