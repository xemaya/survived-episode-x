## 切片设计目标

1. **集内高峰：周日 9:30 KPI Review 浮层揭晓 5 路径** —— 这是 anti-Pillar 1 第一次咬人。无论笑天选哪条路径，下月 threshold 都涨。**没有 Game Over 路径**——S1 是教学集，让玩家学习"做对了反而下月更难"
2. **D Finale 集中触发**——9 个 NPC（除林姐）在 E4 各有 D finale beat，体量从 1 句台词（IT 小马 / 老周）到完整段落（Zoe / 妈妈 / Lisa cliffhanger）
3. **5 : 5 笑泪比**——上半周笑（王总监继续叫不准 + David 周二写下周计划 + Vivian 撤销福利预告 + David 请客试探），下半周扎到底（KPI Review 揭晓 + 4 NPC D finale 反高潮 + Lisa 微信暗示状态下滑）
4. **不要发生的事（红线）**：
   - Lisa **走 / 留** 不能在本集出现（这是 S3 finale = E12）
   - **HR 介入 Lisa**（HR 找她谈话 / 月度面谈 / 试用期评估）不能在本集出现（这是 S2 finale = E8）
   - 林姐不出场（**S3 finale 路径 A 才第一次**）
   - 5 路径**全部 Lisa "过"**——她在 S1 末状态 OK
5. **Series cliffhanger** 至 S2：周日晚 Lisa 微信"下个月开始我可能加班多一点"——她**没说为什么**。S2 第 1 集（E5）开局兑现：王总监第一次单独 cue 她（不是在晨会公开 cue）

**不确定 / 需要 designer review 的场景**：

1. **5 路径浮层 UI 设计** —— spec 没明确说浮层 UI 是什么。我用 ASCII codebox 模拟"系统语言"——这种格式让"系统冷处理" 的感觉很强，但实装时可能需要更适合 Pixi/Tauri 的 layout（emojis/box-drawing chars 在 retro pixel UI 里可能不渲染好）。我的判断是 keep ASCII（spec 也没说不能），但 flag 给 designer
2. **5 路径触发条件具体数值化** —— spec §6 给了 5 路径的"qualitative trigger"（"卷王模式 / 标准达标 + 帮 Lisa" 等）。我在浮层里写了"触发：每天加班 ≥ 3 次 + 帮 David PPT (E2 周一选 A) + 帮 Lisa PPT (E2 周五选 A)..." 这种具体 condition——这是我**推断的 schema**。具体数值阈值（"≥ 3 次" 这个 3）spec 没说。需要 designer 给 hard rule
3. **公式"max(本月达标值, 上月阈值) × (1 + 本月超额比例)"** —— 我在浮层里第一次给出系统公式。spec §6 表里有数字（100→110, 100→105 等）但没明说公式。我从数字反推了公式。如果不对（比如其实是加法），需要 fix
