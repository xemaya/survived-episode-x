## 1. 核心设计原则

### 1.1 概念锁定

**Daily choice ≠ card**——用户在 design pivot 后明确：核心是"**选择**"，不是卡牌抽牌机制。每个 choice 是一个 **micro-event**：

| 元素 | 剧情 event（season-arc 写） | Daily choice（本文件写） |
|---|---|---|
| 体量 | 5-12 行场景 + 2-3 选项 | 1-2 行场景 + 2-3 选项 |
| 影响 | NPC score + series 弧光 | 3 核心属性 + 偶尔 NPC score |
| 触发 | season-arc spec 时间点 | 时间槽空位 + 当前 context filter |
| 频率 | 全 series ~50 个 | 每集每天 1-3 次 |

### 1.2 三属性 ↔ 不可能三角

> **职场不可能三角**：钱多 / 事少 / 离家近——三者**不可兼得**。

| 属性 | 不可能三角 | 范围 | Game Over Trigger |
|---|---|---|---|
| **KPI** | "事"（多 / 少） | 0-200 | < 50（被裁 → 早期 / 中期 / 后期 GO 文案）/ > 150 累积（晋升候选 → 终极 GO"恭喜晋升"） |
| **钱** | 钱（多 / 少） | 0-? | < 4500（房贷断 → 必须辞职 → soft GO"适应不良"） |
| **状态** | 离家近（个人自由 + 身心健康） | 0-100 | < 20（病倒 → 强制休假 → 6 次累积直接辞退） |

3 属性自然 enforce 三角约束——**任何 daily choice 几乎都在 2 选 1 的交换**。例如:
- 高薪低工作量 → 必须远（钱 + 状态损耗）
- 高薪近通勤 → 必须卷（钱 + KPI 损耗状态）
- 轻松近家 → 必须钱少

### 1.3 "有梗有网感"判定

Daily choice 的写作要让玩家"哈哈哈这就是我"或"扎心了"。判定方法：

1. **朋友圈测试**——真正的中文打工人会截图发朋友圈吗？发"哈哈哈"的笑点 = 通过；发"扎心了"的扎点 = 通过；都不发 = 重写
2. **网感检验**——是否引用了真实中文职场流行语 / 段子（钉钉 999+ / 周报水分 / 已收到 / 颗粒度 / 加班餐九宫格 / 朋友圈屏蔽爸妈 / 35 岁焦虑 / 卷王 / 摸鱼 / 摆烂 / 内卷 / 内耗 / 打工人 / 社畜）
3. **笑天 voice 检验**——选择后的笑天内心独白要保持 protagonist.md §4 的"观察者位置 + 自嘲优先 + 不煽情 + 反高潮"

### 1.4 NPC score 影响

部分 daily choices 会影响 NPC score（特别是"NPC 互动"类）。但 daily choice **不该是推 series 弧光的主力**——那是剧情 event 的工作。Daily choice 的 NPC 影响应该是 **flavor 级别的小波动**（±1 到 ±5），不是大决策。

例外：极少数 daily choice 是**剧情 event 的 setup / payoff**（如：你帮 Lisa 拼了 12 次奶茶 → S2 Lisa 状态下滑时她**只会找你倾诉**而不是 David）

## 2. 数值平衡规则

### 2.1 影响幅度 anchor

| 幅度 | 范围 | 例子 |
|---|---|---|
| **轻** | ±1 到 ±3 | 茶水间偷喝同事咖啡 → 钱 +1 / 状态 +2 |
| **中** | ±5 到 ±8 | 周报水分 → KPI +5 / 状态 +3 |
| **重** | ±10 到 ±15 | 周末加班 → KPI +5 / 钱 +20 / 状态 -15 |
| **临界** | ±20+ | 搬家近公司（大决策）→ 钱 -200 / 月 / 状态 +20 / 月 |

每个 choice 影响 **1-3 个属性**（不能 4 个全影响——选择就没意义）。

### 2.2 月末固定结算

| 触发 | 影响 |
|---|---|
| 月末发工资 | 钱 +11000 |
| 月末扣房贷 | 钱 -4500 |
| 月末给妈打钱 | 钱 -1000（玩家可选关闭，但关闭 → mom score 暴跌） |
| 月末 KPI Review | 计算下月 threshold（per series-structure） |
| 月末病倒次数检查 | 累积 > 6 次 → 直接辞退 GO |

### 2.3 周末 / 工作日差异

| 维度 | 工作日 (周一-周五) | 周末 (周六-周日) |
|---|---|---|
| 时间槽 | 8 AP（含上下班）| 自由时间 4 AP（不需要打卡） |
| 状态恢复 | 工作消耗 ~5-15 / 天 | 自然恢复 +20 / 天（睡 11 点起） |
| Daily choice 池子 filter | 含通勤 / 午餐 / 工作内容 / NPC 互动等 | 仅"小确幸 / 大决策 / 存活 buffer / 家庭"类 |
| 妈妈视频 | 不出现 | **周日 8:30 必出现**（per npcs.md §9） |

### 2.4 解锁规则

| 解锁条件 | 解锁内容 |
|---|---|
| **S1-S2 (E1-E8)** | 通勤 / 午餐 / 工作内容 / 小动作 / NPC 互动（基础 5 类）|
| **S3+ (E9-onward)** | 加 "大决策"类（搬家 / 接猎头 / 投简历 / 请年假）。理由：S1-S2 笑天还在适应期，他不会想搬家 |
| **任何状态首次 < 20** | 解锁"存活 buffer"类（病倒后处理 / 焦虑后处理 / 钱紧后处理）|
| **晋升候选 flag = true** | 解锁特殊 choice"主动找王总监谈"（自我加速终极 GO） |
| **接到猎头电话 (S10+)** | 解锁"投简历 / 面试请假"类 |

### 2.5 病倒 / 焦虑机制

**状态 < 20 触发病倒**：
- 立即标记 sick = true
- 下一天 morning_briefing 跳过到中午 12:00（半日强制休假）
- 状态恢复到 50
- 病倒次数 +1
- 当天可触发 daily choice："请病假" / "硬撑去公司" 二选 1

**series 内累积 6 次后**：
- 第 7 次病倒 → 立即终极 GO："小笑啊…陈天啊…我们觉得你可能身体上不太适合这个岗位"
- 提前在第 5 次病倒时触发 warning：HR Zoe 找笑天"陈笑天先生，您本季度病假累积已 5 次"

**焦虑机制（状态高位但 KPI 也高位）**：
- 状态 > 70 + 累积本月加班 > 30 小时 → 触发"焦虑 burst"
- 焦虑 burst：状态突降 -20 + 强制下一天 status mug 是空 + 烟雾
- Per series 触发 cap 不限，但每次 burst 都让笑天"想跳槽"flag +1

## 3. Choice 写作格式（模板 spec）

每个 daily choice 用以下 markdown 格式：

```markdown
### Choice N · [Choice 名] · [类目]

**触发**：[时间窗口 / 前置 flag / context filter]
**速度**：[闪 ~1-2 行 / 标准 ~3-5 行]
**解锁**：[基础 / S3+ / 病倒后 / 等]

> [场景描述 1-2 行——观察者位置，二人称"你"]
>
> [可选：1 行 NPC 行为 / 物的状态]

**[A 选项 ≤ 4 字]** —— [后果 1 行 NPC 行为 / 物状态]  `→ 属性变化`
**[B 选项 ≤ 4 字]** —— [后果 1 行]  `→ 属性变化`
**[C 选项 ≤ 4 字]** —— [后果 1 行]  `→ 属性变化（可选 + 隐藏 flag）`

> _笑天内心独白 1-2 句——观察者位置 / 自嘲优先 / 不煽情_
```

**写作 checklist**（per `tone-bible.md` v2 5 原则 + 4 工艺细节）：
- [ ] 选项 ≤ 4 字 + 不评价 + 不解释
- [ ] 后果 1-2 行 NPC 行为 / 物状态陈述，不评价不煽情
- [ ] 笑天内心独白 1-2 句，先嘲自己再嘲别人
- [ ] 朋友圈测试通过
- [ ] 影响 1-3 个属性，不能全 4 个
- [ ] 至少 1 个网感梗（见 §1.3 列表）

### 5.1 通勤类

#### Choice 01 · 加班 9:30 后打车回家可以报销，但要写申请 · 通勤

**触发**：after_work 选了"申报加班"且时间过 21:30
**速度**：闪 (~2 行)
**解锁**：基础

> 21:30。加班还差 1 分钟才能报销打车费——公司规则是 21:30 之后才报。
>
> 你看了一眼时间。21:29:34。

**[A 9:31 打车]** —— 你多坐了 1 分 26 秒。打车回家，凭加班证明报销 30 块。 `→ KPI +5 / 钱 +30 / 状态 -3`
**[B 9:29 打车]** —— 自费 30 块。早 90 秒下班。 `→ KPI 0 / 钱 -30 / 状态 -5`
**[C 蹭同事车]** —— 张哥说"顺路" 但其实绕了 5 公里。 `→ KPI 0 / 钱 0 / 状态 -1 / 张哥 score +1`

> _我为了 60 块加班 91 秒。这种数学题中年人都会做。_

### 5.3 小动作 / 小确幸类

>

**[A 排]** —— 30 分钟后你拿到一杯。等的时候你看了 5 篇小红书。 `→ KPI -5 / 钱 +18 / 状态 -10`

> _但奶茶是奶茶。_

### 5.4 NPC 互动类
