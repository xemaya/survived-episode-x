## Project State

This is a **salvage** of a Godot 4.6 / GDScript project (《活过第 X 集》— a pixel-art reverse-KPI office survival sim). Detailed framing is in `README.md`. Two facts dominate every other instruction:

1. **`project.godot` does not exist yet.** `src/`, `assets/`, `addons/` are real code/data, but Godot has never been pointed at this tree. Before any runtime claim ("tests pass", "scene loads", "X works"), a fresh Godot 4.6 project must be created and these directories imported.
2. **Code is "paper-done."** ~18.6k lines of GDScript in `src/` and ~41k lines of test code in `tests/` were generated by an autopilot system, then committed without ever being run through Godot. README's "真东西 vs paper-done" table is the honest map of which systems have substance vs which are thin. Treat any function as suspect until proven by execution.

The previous framework's ceremony (epic/story status, `Status: Complete`, ADR governance gates, autopilot/agent/skill workflow) has been stripped from the workflow but the *artifacts* remain. Read them as design notes, not as a running process.

- **Engine:** Godot 4.6 (2026-01 release) + GDScript, Forward+ renderer, Godot Physics 2D.
- **LLM training cutoff vs engine:** Godot 4.4 / 4.5 / 4.6 are post-cutoff. Verify any non-trivial engine API against `docs/engine-reference/` (per `src/CLAUDE.md`) before using it. Don't guess `@abstract`, `duplicate_deep()`, dual-focus, AgX tonemapper, CSV plural form, etc.

## Common Commands

All commands assume a `project.godot` has been created at the repo root and `src/`, `assets/`, `addons/` are imported.

```bash

# Run tests in one directory

# Run a single test file
```

## The 5 Red Lines (architecture/architecture.md §Architecture Principles)

These govern every code/design decision and are enforced by lint chains, not just policy:

1. **Pillar 4 anti-hero (`subject_inversion_lint.py`, 8 domains):** No "你做到了 / 完美 / 战略 / 友谊" tone in `EVENT/NPC/AP/KPI/EFFORT/TENURE/RECAP/TUTORIAL` strings. Use `_IRONY` and `_BUREAUCRATIC` suffixes to whitelist HR-sneer copy. PR-blocking.
2. **Anti-Pillar 1 monotonicity:** AP cap, `monthly_threshold`, and `capacity_factor(m)` are monotonic in one direction (8 cap fixed, threshold up-only, capacity down-only). Any effect/event/unlock/setting that reverses this fails CI with `push_error`. Run Meta unlocks are content-only across 5 whitelisted classes (`codex / memo / npc / event_branch / ending`) — no mechanical growth fields.
3. **Diegetic UI lock (art-bible §7.1):** No floating screen-space HUD during `ACTION_DAY / EVENT_ACTIVE / WEEKEND / MAIN_MENU`. All info embedded in workstation props (sticky notes, coffee mug, monitor, calendar, NPC sprite/position, empty chair). `CanvasLayer.visible = true` is allowed only in `PAUSE / KPI_REVIEW / GAMEOVER / SETTINGS`. `#19` notifications are diegetic-element variants, never popups.
4. **`#6 SceneDayFlow` is the single dispatch owner:** `scene_state_changed` has exactly one emitter; downstream systems must not self-drive sub-mode transitions. `request_transition()` is the only legal entry. `accumulation_event` is similarly single-owner (`#5 Lighting` only).
5. **Data-driven + engine-contract lock:** Numeric tuning lives in `config/*.tres` and `design/registry/entities.yaml` (13 named constants — do not duplicate values inline). Godot 4.6 specific API usage is centralised; check `architecture/control-manifest.md` `Forbidden APIs (Godot 4.6)` for the deprecation table.

`architecture/control-manifest.md` is the single-page programmer reference distilled from ADR-0001..0017 — read it before touching Foundation/Core code.
